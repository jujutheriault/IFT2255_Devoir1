# Glossaire

Ce glossaire définit les termes importants utilisés dans le projet, en particulier ceux qui pourraient prêter à confusion.

| Terme | Définition |
|-------|------------|
| Étudiant | Désigne un utilisateur principal de l'application. Étudiant et utilisateur sont des termes interchangeables.  |
| Éligibilité | On parle d'éligibilité lorsqu'un utilisateur doit vérifier s'il possède les préalables nécessaires au cours sélectionné. |
| Profil | Un profil contient les information qui sontdemandées à l'utilisateur et qui seront utilisées pour l'usage de l'application. |
| Cours pratique | On dit d'un cours qu'il est pratique si l'emphase du cours est sur la mise en pratique de la théorie apprise dans un ou plusieurs cours.Par exemple, un cours de programmation qui demande l'utilisation de la théorie des mathématiques et des structures de données mais ne mets l'emphase sur l'apprentissage de ces notions. |
| Cours théorique  | Se dit d'un cours où l'accent est mis sur l'apprentissage de la théorie jugée utile pour la formation d'un programme. Par exemple, des cours de mathématiques sont nécessaires à plusieurs programme pour comprendre certains aspects de la matière. |
| Programme | Un programme est un ensemble de cours qui donne accès à un diplôme. Un programme regroupe généralement des cours propres à une discipline spécifique à quelques cours au choix près. Les programmes constituent aussi différents niveux de formation(baccalauréat,maîtrise,doctorat,certificat,etc.). |
| Cours | Ressource qui est manipulée par l'application pour les utilisateurs |
| Crédit | Un crédit est une unité de mesure de la charge de cours et des heures de cours allouées. Un nombre de crédit à atteindre est nécessaire pour réussir un programme et obtenir un diplôme. |
| Difficulté perçue | Difficulté d'un cours basé sur les avis étudiants. Comment les étudiants percoivent la difficulté d'un cours dépendamment de leur expérience avec les travaux, les heures de cours et les professeurs. |
| Charge de travail | Quantité de temps et d'efforts à consacrer pour réussir un cours. Une charge de travail peut contenir les cours, les travaux pratiques et les heures à travailler sur le cours. Une mesure de la charge de travail approximative est donnée par l'université et les avis des étudiants. |
| Cheminement | Regroupement des cours nécessaires pour l'obtention d'un diplôme dans un programme donné. Certains programmes ont des cheminements types,comme les baccalauréats, alors que d'autres n,en on pas, comme les micro programmes ou les mineures.|
| Cours hors programme | Un cours qui n,est pas dans le cheminemtn type d'un programme. Certains de ces cours peuvent êtres crédités pour l'obtention d'un diplôme alors que d'autres non. |
| Conflit d'horaire | Situation survenant lorsque les heures des horaires de deux ou plusieurs cours se chevauchent. Un étudiant ne peut donc pas être présents pour ces deux cours simultanément. Un horaire en bonne et due forme ne contient pas de conflits. |
| Avis étudiant | Message laissé sur discord qui peut être utilisé pour la plateforme si elle répond au critères de pertinence. |
| Cours préalable | Cours à réussir pour pouvoir choisir et entreprendre un autre cours.Le cours préalable est indiqué dans la description des cours. |
| Orientation | branche spécifique d'un programme qui contient des cours axée sur une spécialisation d'une discipline précise qui peut être soit technique ou théorique. |
| Résultat agrégé | Résultats officiels des cours de l'université de Montréal. Les résultats agrégés contiennent la moyenne d'un cours, le nombre d'inscrits et le nombre d'échecs. |
